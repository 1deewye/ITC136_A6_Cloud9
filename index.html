Introduction and Elements of the stack

Node.JS

Node.js was created by Ryan Dahl in 2009. Dahl's goal was to make JavaScript, primarily a client-side language, run in back-end development. By 2010 Node had already become popular enough that the Node Package Manager, or NPM, was created in order to help people run, and pass on, Node based tools and libraries. Over the years, as programmers worldwide try to simplify their own lives, Node has become a major building block in the “JavaScript Everywhere,” movement. This idea states that all web app development should revolve around the same coding language, rather than running many different things server side.

Express.JS

Express is a backend part of many stack. Express can be described as middleware or what is between the server and the client. Express creates an exceptionally fast connection between the client and the server when paired with Node.js. Express is easy to test as well, because it is between the two. Express has a wide range of functions. It has successfully used the NPM making it so people can customize express based on what third party plugins they choose. Express has a lot of community support for it because it handles things like cookies, routes, and requesting body and forming the response, it is usually better for small to medium projects. Overall, express is a minimal bundle of features and functions that make it easier to build web apps.

AJAX, Routing and CRUD

AJAX is a set of Web development techniques using many Web technologies on the client side to create asynchronous Web applications. AJAX stands for Asynchronous JavaScript And XML. AJAX is not a programming language. It uses a combination of a browser built-in XMLHttpRequest object and JavaScript and HTML DOM. AJAX allows web pages to be updated asynchronously by exchanging data with a web server behind the scenes. This means it's possible to update parts of a web page, without reloading the whole page.

How To

As with most things, the best way to really get a good grasp on something is to playt around with it. In the following demo I'm going to show you how to make a super simple, super basic "Hello World" app in Cloud 9, from AWS

Your standard "Hello World" project is simply a way to design, from the ground up, the most simple and fundamental version of a project, and have it display the terms "Hello World" where it is supposed to. Basically it creates the barebones, then runs it to make sure it works. Once you have the barebones you can play around with it some more in order to make it fancier.

  1. Spin up the most simple workspace in Cloud 9

    The first thing we're going to do is spin up a basic workspace. For this tutorial we're actually going to avoid the Node.JS workspace that Cloud 9 provides due to the fact that it contains a bunch of stuff that we're not currently interested in. For now, just go ahead and spin up one of the Ubuntu workspaces. This will create a basic command line for you to work with.

  2. Install your dependencies

    Once you have your command line ready to go, we're going to install the pieces of the puzzle for us to work with. Cloud 9 is already built using Node.JS, so that bit is taken care of. We do want ot install Express.JS, which is the middleware that our front end will be using.

    • Use the npm init command to install a package.json file in your program.

      npm init

      You can accept all the defaults, but change the name of your "entry point" from the default to "app.js"

    • Now install and save Express into the myApp directory.

      npm install Express --save

      This will install Express and save it into your dependencies

  3. Set up the requirements in your main file

    In the app.js file, use the save the following commands to the top of the file.

    var express = require('express');
    var app = express();

  4. Set up your listening port

    Now you'll want to tell the program where the output of the file is going to be shown. Cloud 9 has special variables for this to ensure that it stays inside the develpment environment.


    app.listen(process.env.PORT, process.env.IP, function() {
    console.log('This app is listening on port 3000')
    });

  5. Set up the functionality

    Alright! So now we've got the program importing all the things we need for it to run, and we've got a designated space where the output is going to be shown. Now we need to create the program itself.

    app.get('/', function(req, res){
    ]res.send('Hello World')
    });

    What we've done here, step-by-step, is to tell the app to use the express middleware (app.get()) to send a get request'/', function(req, res) to the root level of our dummy web page. Once we have the connection, the webpage will send a responseres.send("Hello World") of the term "Hello World" that will render on the browser window.

  6. Run the Code

    In the terminal for the app.js file you'll see a button to run the program. Click the button and then click the link for the web page that is provided to you. You should see the results of your work there.

  7. Mess around with what you've built

    Now that you have the functionality built, you can play with it. Try changing the output of the respoese (res.send("")), or change where the output is going to be displayed (change the location from '/' to 'wherever'). You should be able to check the page by typing your new destination on the end of the browser path provided to you.
